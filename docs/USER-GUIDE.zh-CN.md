# tamagotchi-smart-dim-flasher 使用文档

这份文档面向第一次使用本工具的用户，重点说明启动方式、页面按钮含义、推荐操作流程，以及常见报错的处理方法。

## 适用范围

- 操作系统：`macOS`
- 程序器：`CH341A`
- 目标卡：`Tamagotchi Smart` 相关 `DiM / TamaSma / BIM`
- 刷写后端：`flashrom`
- 交互方式：本地 `Web UI`

本项目主要按 `Tamagotchi Smart` 场景整理和验证。其他同类卡片可能兼容，但请自行确认。

## 启动前准备

请先确认下面几项：

- 已安装 `Python 3.10+`
- 已安装并可执行 `flashrom`
- 已连接 `CH341A`
- 已把目标卡插到程序器上
- 已准备好要写入的 `BIN`

如果你使用 `Homebrew`，通常可以这样安装 `flashrom`：

```bash
brew install flashrom
```

## 启动 Web UI

在仓库目录执行：

```bash
./start-web-ui.command
```

首次启动会自动完成下面几件事：

- 创建 `.venv`
- 安装 `requirements.txt` 里的依赖
- 启动本地 Web 服务

正常情况下，终端会打印类似这样的地址：

```text
DiM tool web UI: http://127.0.0.1:8765
```

浏览器会自动打开。如果没有自动打开，也可以手动访问终端里显示的地址。

如果遇到以下权限报错：

- `LIBUSB_ERROR_ACCESS`
- `root privilege`
- `Cannot detach the existing USB driver`

请改用：

```bash
sudo ./start-web-ui.command
```

也可以直接双击：

- `start-web-ui.command`
- `start-web-ui-root.command`

## 页面按钮说明

首页的主要操作都在第一屏，建议先理解每个按钮的作用。

### `上传并备份当前卡`

- 选择一个 `BIN` 后点击
- 工具会先把文件保存到 `runtime/uploads/`
- 然后自动读取当前卡，备份到 `runtime/backups/`
- 备份成功后，页面会把这个 `BIN` 记为“当前上传文件”

### `安装到当前 DiM 卡`

- 使用“当前上传文件”作为写入来源
- 工具会再次备份当前卡
- 然后根据 `BIN` 类型生成最终待写入镜像
- 最后执行写入和校验

### `仅备份当前卡`

- 只读取当前卡并生成备份
- 不会改动卡内容
- 适合先留一个干净备份再做其他操作

### `探测当前卡容量`

- 调用 `flashrom` 识别当前卡芯片和容量
- 页面会显示最近一次探测到的芯片型号与容量
- 建议每次更换卡后都先点一次

### `重置当前卡使用次数`

- 会先备份当前卡
- 然后扫描卡内有效 `Tama Smart` 头
- 清空 usage 槽并重算校验
- 如果支持，还会尝试启用整片写保护，减少后续再次累计次数的可能

### `复原最近备份`

- 直接把“最近备份”重新写回当前卡
- 适合把卡恢复到安装前状态

## 推荐操作流程

第一次使用时，建议按下面顺序操作：

1. 连接 `CH341A`，插入目标卡。
2. 启动 Web UI。
3. 点击 `探测当前卡容量`，确认芯片与容量识别正常。
4. 点击 `仅备份当前卡`，先保留一份手动备份。
5. 点击 `上传并备份当前卡`，选择你要写入的 `BIN`。
6. 检查页面里“当前上传文件”和“最近备份”是否已更新。
7. 点击 `安装到当前 DiM 卡` 完成刷写。
8. 如果效果不对，点击 `复原最近备份` 回退。

如果你只是想清掉次数，不想换内容：

1. 点击 `探测当前卡容量`。
2. 点击 `重置当前卡使用次数`。
3. 查看“最近操作”和日志，确认是否已经成功启用写保护。

## BIN 文件规则

本工具目前支持两类输入文件：

### `1MB BIN`

- 这是 `payload` 模式
- 只覆盖前 `1MB`
- 其余区域保留当前卡备份中的内容
- 适合只替换主要数据而保留卡的其余布局

### 完整 `BIN`

- 文件大小必须和当前卡真实容量一致
- 当前支持范围为 `2MB` 到 `16MB`
- 工具会整片写入

如果文件既不是 `1MB`，也不在 `2MB-16MB` 的完整镜像范围内，页面会直接拒绝。

## 页面状态区怎么看

页面下半部分会持续显示最近一次操作结果，主要可以看这几块：

- `当前上传文件`
  显示最近导入的 `BIN` 文件名、大小、时间和 `SHA-256`
- `最近备份`
  显示最近一次自动或手动备份的文件信息
- `最近操作`
  显示最近一次刷写、重置 usage 或复原的结果摘要
- `运行状态`
  显示 `flashrom` 路径、程序器类型、探测结果、运行目录，以及完整日志

如果某一步失败，优先看最下方日志区，通常能直接定位是权限、芯片识别还是文件大小问题。

## 运行目录和生成文件

运行过程中，工具会在仓库内生成这些内容：

- `runtime/uploads/`
  保存你通过页面上传的 `BIN`
- `runtime/backups/`
  保存自动或手动生成的卡备份
- `runtime/prepared/`
  保存实际写入前生成的中间镜像
- `runtime/state.json`
  保存页面最近一次状态

建议不要手动删除正在使用中的文件；如果要清理，最好在工具关闭后再操作。

## 常见问题

### 1. 页面提示 `flashrom` 不可用

通常表示系统里找不到 `flashrom`。先确认：

- `brew install flashrom` 已经执行成功
- `flashrom` 可以在终端里直接运行

你也可以手动指定路径：

```bash
DIM_FLASHROM_BIN=/opt/homebrew/bin/flashrom ./start-web-ui.command
```

### 2. 提示权限不足或 USB 设备无法接管

常见关键词包括：

- `LIBUSB_ERROR_ACCESS`
- `root privilege`
- `Cannot detach the existing USB driver`

处理方式：

1. 关闭可能占用 `CH341A` 的其他软件。
2. 重新插拔一次程序器。
3. 用 `sudo ./start-web-ui.command` 重新启动。

这类报错通常是权限问题，不一定是卡内容损坏。

### 3. 提示上传的完整 BIN 和当前卡容量不匹配

这表示：

- 当前卡实际容量和你的完整镜像大小不同
- 或你上传的不是 `1MB payload`

先点一次 `探测当前卡容量`，确认当前卡是 `2MB / 4MB / 8MB / 16MB` 中的哪一种，再换对应大小的镜像。

### 4. 重置使用次数时报“没有找到有效的 Tama Smart 头”

这通常表示当前卡镜像里没有识别到可处理的 `Tama Smart / TamaSma` 数据头。可能原因包括：

- 当前卡并不是这类卡
- 当前卡内容不是有效镜像
- 卡内容已经损坏

这种情况下不要继续反复重置，建议先做一次完整备份并检查镜像来源。

### 5. 重置成功了，但写保护没有成功启用

这意味着 usage 已经清零，但后续仍可能再次累计。常见原因是：

- 当前芯片不支持对应的写保护命令
- `flashrom` 对该芯片的写保护支持有限

这不一定影响本次写入成功，只是“防再次累计”的保护没有完全到位。

### 6. 端口被占用，页面打不开

程序默认从 `127.0.0.1:8765` 开始尝试端口。如果本机这个端口段被占用，可以手动换端口：

```bash
DIM_PORT=9000 ./start-web-ui.command
```

然后访问终端打印出来的实际地址。

## 可选环境变量

如果你需要调整启动参数，可以在命令前临时设置环境变量：

```bash
DIM_PORT=9000 DIM_OPEN_BROWSER=0 ./start-web-ui.command
```

支持的常用配置包括：

- `DIM_PORT`
  指定 Web UI 端口
- `DIM_HOST`
  指定绑定地址，默认 `127.0.0.1`
- `DIM_OPEN_BROWSER`
  设为 `0` 时不自动打开浏览器
- `DIM_FLASHROM_BIN`
  指定 `flashrom` 可执行文件路径
- `DIM_PROGRAMMER`
  指定程序器，默认 `ch341a_spi`
- `DIM_CHIP`
  指定默认芯片定义，默认 `MX25L3205(A)`
- `DIM_FLASHROM_TIMEOUT`
  指定 `flashrom` 超时时间，单位秒
- `DIM_MANAGE_WRITE_PROTECT`
  设为 `0` 时跳过写保护管理

## 本地资源目录

仓库里的 `tamasmart/` 目录适合存放你自己的：

- 卡备份
- 完整镜像
- 其他本地资源

这个目录默认不会进入 Git，适合做个人资料库使用。

## 风险说明

请在使用前明确这些风险：

- 刷卡和写保护操作都涉及真实硬件
- 错误镜像可能导致卡内容不可用
- 非 `Tamagotchi Smart` 场景下兼容性没有完整验证

最稳妥的习惯是：

1. 每张卡第一次操作前先手动备份。
2. 每次更换卡后先探测容量。
3. 只写入来源明确的镜像。
